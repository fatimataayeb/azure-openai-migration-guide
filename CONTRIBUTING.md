# Contributing to the Azure OpenAI Migration Guide

Thank you for your interest in contributing! This guide helps organizations migrate from GPT-4o to GPT-5.1, and community contributions make it better for everyone.

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion:

1. Check existing issues to avoid duplicates
2. Open a new issue with:
   - Clear description of the problem or suggestion
   - Steps to reproduce (for bugs)
   - Your environment details (Azure region, SDK version, etc.)

### Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Test your changes
5. Submit a pull request

### What We're Looking For

- Bug fixes in scripts or documentation
- Additional golden dataset examples
- Improvements to evaluation pipelines
- Documentation clarifications
- Translations (especially Arabic)
- Real-world migration experiences

### Code Style

- Python: Follow PEP 8
- JavaScript: Use ESLint with standard config
- Documentation: Clear, concise, customer-friendly language

## Code of Conduct

- Be respectful and inclusive
- Focus on constructive feedback
- Help others learn

## Questions?

Open an issue with the "question" label or reach out to the maintainers.
